GitBash 自带 vim 无法加载 python 库的问题


阅读数: 次    2019-12-08

问题说明

Windows10 系统下，使用 GitBash 自带的 Vim 编辑器，在安装插件时发现 Vim 无法加载 Python 库
导致安装的 vim 插件无法正常运行

解决方法

1.vim 输入：help python-dynamic 确定 vim 所需 python 库版本

如图

我们可以看到，该版本 vim 支持两个版本 python 库，分别对应
pythondll : python27.dll
pythonthreedll : python35.dll

2. 下载相应版本的 python，并设置环境变量 Path

1.python 3.5 我下载的是 zip 解压版
2.python 2.7 没有 zip 版本，安装版在安装后安装目录可能会缺少 python27.dll 文件，这时不要尝试手动去添加它，卸载了重新安装或者重新下载安装包。

3. 配置 vim 配置文件

在.vimrc 中配置
set pythonthreedll=c:/Python35/python35.dll
set pythondll=C:/Python27/python27.dll
这里的地址按你自己的地址来

不出意料这时你的 python 库就设置好了，一般来说，这时如果还存在 python27 库无法加载的问题，多半是 python2.7.X 细化版本的问题，请换版本多次尝试
