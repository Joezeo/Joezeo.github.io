JAVA 笔记：Stream 流机制


阅读数: 次    2020-01-24

Stream 简介

stream 是在 Java8 中添加的新特性，stream 是用于对集合迭代器的增强，使之可以更高效地完成集合操作（过滤、排序、分类统计等等），通常与 lambada 表达式结合使用。

示例代码

Lambda 表达式

lambda 表达式是一个可传递的代码块，可以在以后执行一次或多次。

简而言之，lambda 表达式是一种简化接口方法重写的书写形式

lambda 表达式的使用条件：接口中只有一个方法，不包括接口默认实现 (JDK1.7 及以后）的方法，Object 类定义的方法。
符合这样的接口有被称为函数式接口，可用 @FunctionalInterface 注解对接口进行判断是否是函数式接口。

lambda 表达式基本表达形式 : 参数 -> 表达式

为什么使用 Stream 流

传统的 JavaEE 项目中，数据来源很大程度上依赖关系型数据库，如 Mysql、Oracle 等等，对于数据的处理可以通过 sql 语句来统一完成。
现如今的互联网项目中，数据来源多样如 Reddis、NoSql、ElasticSearch 等等，这时就需要对多个数据来源的数据进行处理，传统的方式通过循环遍历处理数据，不仅繁琐并且可读性差。
Java8 中加入了 Stream 流机制，使得可以更加方便地处理数据，代码的可读性也更高。

Stream 使用流程示意图

Stream 的特点

  1. 流不会改变原数据源的数据
  2. 流不会在内存中储存数据
  3. 流不可重复使用（一旦产生新的流，旧的流就会关闭）

流的两种操作方式

  1. 中间操作：惰性的，只有在执行终值操作前执行
  2. 终值操作：只能有一次，并且在最后
